# 根目录职责治理

最后更新：2026-03-29
状态：现行治理说明
适用范围：当前仓顶层目录与顶层职责变更

## 一句话结论

顶层目录不是随手扩的缓存区；若确需新增顶层目录，必须先写清职责、边界、启用条件与回退口径，再修改锁定清单。

## 当前顶层职责

- `docs/`：现行文档与正式规则
- `.codex/`：运行态任务包与维护层脚手架
- `logs/`：本地留痕，不进入公开提交
- `temp/`：生成物与历史快照
- `codex-home-export/`：当前仓本机生产母体真源，仅承载单机接管最小闭环所需文件

## 顶层新增治理规则

1. 未获人工批准，不得新增顶层目录。
2. 已获人工批准后，仍需先在本文记录职责，再更新 `docs/30-方案/02-V4-目录锁定清单.md`。
3. 新顶层目录必须有唯一职责，不得与既有目录职责重叠。
4. 新顶层目录若承载安装母体、发布母体或迁移母体，必须额外写清：
   - 真源文件
   - 运行态镜像位置
   - 安装/切换脚本
   - 回滚口径
   - 验板步骤

## 当前已批准新增位

- `codex-home-export/`：已批准作为“本机生产母体”顶层位；当前仅允许落最小闭环骨架，未完成切换、回滚与验板前，不视为已正式接管生产。
