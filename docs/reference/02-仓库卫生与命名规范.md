# 仓库卫生与命名规范

## 一条总纪律

**先分类，再落盘；任务结束前必须收口。**

## 根目录准入

根目录只允许长期保留：
- `README.md`
- `AGENTS.md`
- `.gitignore`
- 少量真正必须放根的顶层配置

原则上，其他内容都不应直接长在根目录。
目录结构是否可改，以 `docs/30-方案/02-V4-目录锁定清单.md` 为准；未获人工拍板前不得新增目录。

## 目录落点

- 正式文档：`docs/`
- 运行态任务包：`.codex/`
- 留痕日志：`logs/`
- 临时生成物：`temp/generated/`
- 历史归档：`temp/legacy/`

## 命名规则

- 文档：使用明确语义，必要时加数字前缀，如 `01-...md`
- 决策：统一用 `ADR` 或“决策”命名，不用模糊标题
- 方案：显式写出状态，如 `Trial`、`Target`、`草稿`
- 临时快照：`<来源>-snapshot-YYYYMMDD`
- 历史归档：`<主题>-YYYYMMDD`

## 禁止事项

- 禁止 `最终版`、`临时版`、`new2`、`test1` 这类无语义命名
- 禁止把旧实验、日志、缓存、生成物长期放根目录
- 禁止把“以后再整理”当成默认做法

## 交付前收口清单

- 新增文件是否放对目录
- 临时文件是否已迁走
- 历史材料是否进了 `temp/legacy/`
- 生成物是否进了 `temp/generated/`
- 文档索引是否需要同步更新
