# V4 本机生产切换最小闭环方案

最后更新：2026-03-29
状态：子线推进稿
适用范围：当前仓接管本机生产母体前的最小闭环设计

## 一句话结论

当前仓要想丝滑接管本机生产，不能直接整包搬旧母体；最小正确做法是先形成“生产母体职责 + 版本真源 + 切换/回滚 + 验板闭环”的四件套。

## 当前现状

1. 本机运行态版本镜像在 `~/.codex/config/cx-version.json`。
2. 当前镜像仍声明真源为 `codex-home-export`。
3. 旧参考仓仍持有现役 `codex-home-export/`。
4. 当前新仓已落地 `codex-home-export/`、`VERSION.json`、`install-to-home.ps1`、`rollback-from-backup.ps1` 与 `verify-cutover.ps1`，并具备自动验板闭环；仍建议保留面板人工验板。

## 本线最小目标

- 让当前仓具备“单机本机生产切换”能力。
- 切换后可通过版本镜像、安装脚本与固定验板步骤确认是否成功。
- 失败时可一键回滚，不碰用户凭据与隐私文件。

## 明确不做

- 不做第二机链路。
- 不做发布链或便携包链路。
- 不做整包照搬旧母体。
- 不在当前轮引入重型安装器重构。

## 最小闭环四件套

1. `codex-home-export/`：当前仓内的本机生产母体目录。
2. `VERSION.json`：母体版本真源。
3. `install-to-home.ps1` + 回滚脚本：负责单机切换与恢复。
4. 切换后验板清单：确认版本、入口命令与面板表现。

## 验收口径

- 不覆盖用户凭据文件。

- 切换失败可回滚。
- 切换后版本来源、切换来源与验板结果可追溯。
