# V4-Target 冻结清单

最后更新：2026-03-29
状态：现行冻结清单
适用阶段：`V4-Target` 执行前
说明：本清单用于冻结当前 Target 阶段的做、不做、先后顺序与边界，避免执行期口径漂移

## 一句话结论

`V4-Target` 当前只冻结“口径、顺序、边界”，不冻结未来全部实现细节。

## 当前必须冻结的事项

1. 当前仓仍是唯一现行准备落点。
2. 官方面板仍是默认入口。
3. 任务包继续作为唯一运行态目标结构。
4. 维护层动作继续保留在当前目录内自含实现。
5. 执行区现行件与归档区边界继续有效。

## 当前不进入的事项

1. 不直接重构安装链。
2. 不直接扩独立 UI。
3. 不直接上重型自动化。
4. 不直接整包迁入旧仓资产。
5. 不直接推翻 Trial 文件体系重做。

## 当前优先顺序

1. `T1`：冻结 Target 口径
2. `T2`：稳定维护层动作
3. `T3`：固化归档节奏
4. `T4`：准备更强工程化能力
5. `T5`：再决定是否进入更大范围实现

## 当前验收点

- 是否继续保持当前目录内自含
- 是否继续保持现行件与证据稿边界清楚
- 是否继续保持任务包可恢复、可追溯
- 是否继续减少维护层重复劳动

## 触发重新拍板的情况

- 若要改动已锁定目录结构
- 若要引入独立 UI 或重型自动化
- 若要整体迁入旧仓资产
- 若要改变当前默认入口或运行态目标结构

## 本稿结论

`V4-Target` 当前最重要的不是加速扩面，而是守住已冻结口径，按顺序推进。
