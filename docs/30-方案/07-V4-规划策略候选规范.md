# V4-规划策略候选规范

最后更新：2026-03-29
状态：现行方案候选件
适用阶段：`V4-Target` 的 `T4` 准备阶段
说明：本文件用于把“科研型规划策略”收成当前仓内唯一候选口径，先固定策略抽象、输入输出、调度时机与边界，不提前启动新目录或对应实现。

## 一句话结论

`V4` 后续若要增强“先判断、再分步、再验证、再修正”的能力，应优先落成一套轻量规划策略，而不是先上浏览器执行器、多代理或独立 UI。

## 当前定位

- 它不是新模块落地说明，而是当前仓内的策略候选规范。
- 它服务于任务包控制平面，不替代任务包 5 件套。
- 它的价值在于减少高不确定任务中的盲推进与返工。

## 为什么先做这条线

1. 与当前“最小推进点 + 可验证 + 可回溯”主线同向。
2. 对复杂任务的长期复利高于执行器扩面。
3. 不要求改目录、不要求改入口、不要求引入外部依赖。
4. 能直接复用现有 `contract.yaml`、`state.yaml`、`decision-log.md`、`result.md` 作为落点。

## 目标问题

本策略要解决的不是“怎么执行网页”或“怎么多代理协作”，而是下面四个问题：

- 当前任务的核心假设是什么。
- 下一步该拆成哪几个最小可验证动作。
- 每一步的验证信号是什么。
- 若验证失败，应该回退、改路还是暂停。

## 最小策略闭环

### 1. 假设生成

- 先写出当前任务最关键的推进假设。
- 假设必须能被后续证据验证，而不是空泛口号。

### 2. 步骤拆解

- 把任务拆成 1 到 3 个最小可验证动作。
- 每一步都必须说明“完成后能证明什么”。
- 若任务信息不足，优先缩短步长，不强行铺大计划。

### 3. 验证循环

- 每一步完成后，都要有最小验证动作。
- 验证通过，才允许进入下一步。
- 验证失败，必须显式写入失败原因与下一次修正方向。

### 4. 路线修正

- 若失败属于信息不足，优先补信息。
- 若失败属于边界冲突，优先形成拍板输入。
- 若失败属于执行异常，优先进入异常或回退链路。

### 5. 收口更新

- 把结论写回 `decision-log.md` 与 `result.md`。
- 必要时同步 `state.yaml` 的主状态与下一步。
- 保证每轮结束后，别人能从任务包继续接手。

## 推荐输入

- `contract.yaml`：目标、边界、验收、默认自动动作。
- `state.yaml`：当前主状态、风险等级、下一步。
- `decision-log.md`：已有决策依据。
- `result.md`：当前进度、遗留项、下一步建议。
- `gates.yaml`：是否存在待拍板事项。

## 推荐输出

- 一条当前主假设。
- 一组最小推进步骤。
- 每一步对应的验证信号。
- 失败后的改路条件。
- 本轮完成后的收口口径。

## 与现有控制平面的关系

- 不新增新的运行态目录。
- 不新增新的任务包核心文件。
- 默认只作为任务包 5 件套的“上层思考规则”。
- 若未来需要增强，也优先写成脚手架或模板，而不是重起体系。

## 调度时机建议

### 适合启用

- 任务不确定性较高。
- 当前存在多条可能路径，需要先选最小验证路线。
- 当前已经出现返工风险，继续直推会放大成本。

### 不必启用

- 任务已是低风险机械动作。
- 当前已有明确单步维护模板可直接执行。
- 当前阻塞本质是待拍板或待环境修复，而不是缺规划。

## 与其他候选能力的先后顺序

1. 规划策略候选：优先。
2. 治理审计候选：次优先。
3. 浏览器执行器：后置。
4. 多代理协议：后置。
5. 独立 UI：更后置。

## 当前硬边界

- 不把本规范直接展开成 `/v4/planners/...` 新目录。
- 不因为引入规划策略，就新增长期记忆、多代理或浏览器执行实现。
- 不改变当前“官方面板 + 当前目录内自含 + 任务包 5 件套”的主结构。

## 若未来进入实现，建议最小落法

1. 先做文档模板或脚手架增强。
2. 第一层已落到 `docs/40-执行/01-任务包规范.md` 与 `docs/40-执行/02-任务包模板.md` 的规划骨架增强。
3. 第二层已落到 `.codex/chancellor/create-task-package.ps1` 与 `docs/40-执行/11-任务包半自动起包.md` 的脚手架增强。
4. 再做任务包内的最小半自动规划辅助。
5. 最后才评估是否值得升级为独立能力层。

## 当前验收口径

- 是否仍然保持当前目录内自含。
- 是否没有引入新目录、新依赖、新入口。
- 是否能直接映射到现有任务包文件。
- 是否优先减少盲推进，而不是新增维护负担。

## 本稿结论

当前阶段最值得继续投资的高复利方向，是把规划能力先收成轻量候选规范；这样既不破坏冻结边界，又为后续更强工程化能力留下稳定入口。
