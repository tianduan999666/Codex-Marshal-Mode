## 阅读顺序建议

1. 先看 `docs/30-方案/03-V4-MVP边界清单.md`
2. 再看 `docs/40-执行/01-任务包规范.md`
3. 再看 `docs/40-执行/02-任务包模板.md`
4. 进入试跑前看 `docs/40-执行/03-面板入口验收.md`
5. 需要判断执行区文件角色时，看 `docs/40-执行/04-执行区现行件与证据稿说明.md`
6. 需要验证跨轮恢复时，看 `docs/40-执行/05-跨轮恢复说明.md`
7. 需要看恢复演示时，看 `docs/40-执行/06-跨轮恢复样本.md`
8. 需要拍板 Trial 是否通过时，看 `docs/40-执行/07-V4-Trial-验收报告.md`
9. 需要看缺陷优先级时，看 `docs/40-执行/08-V4-Trial-缺陷清单.md`
10. 需要看后续改造顺序时，看 `docs/40-执行/09-V4-Trial-改进建议.md`
11. 需要处理本地提交稳定性时，看 `docs/40-执行/10-本地安全提交流程.md`
12. 需要更快起包时，看 `docs/40-执行/11-任务包半自动起包.md`
13. 需要看是否正式进入 Target 时，看 `docs/20-决策/02-V4-Target-进入决议.md`
14. 需要看 Target 目标态时，看 `docs/30-方案/04-V4-Target-蓝图.md`
15. 需要看 Target 当前冻结边界时，看 `docs/30-方案/05-V4-Target-冻结清单.md`
16. 需要看 Target 推进顺序时，看 `docs/40-执行/12-V4-Target-实施计划.md`
17. 需要执行维护层动作时，先看 `docs/40-执行/13-维护层总入口.md`
18. 需要判断维护动作类型或做提交前收口时，看 `docs/40-执行/14-维护层动作矩阵与收口检查表.md`
19. 需要向主公准备标准拍板包时，看 `docs/40-执行/15-拍板包准备与收口规范.md`
20. 需要半自动生成首个待拍板事项时，看 `docs/40-执行/16-拍板包半自动模板.md`
21. 需要把拍板结果回写并恢复推进时，看 `docs/40-执行/17-拍板结果回写模板.md`
22. 需要记录失败、回退或待恢复状态时，看 `docs/40-执行/18-异常路径与回退模板.md`
23. 需要裁决多 gate 或多异常的主状态时，看 `docs/40-执行/19-多 gate 与多异常并存处理规则.md`

## 使用原则

- 先看本总览，再按固定顺序读取现行标准件。
- 只有现行标准件可以作为当前口径继续扩展。
- 若现行标准件与时间戳稿有差异，以现行标准件为准。
- 若后续新增新的现行标准件，应先更新本总览，再更新首页与导读入口。
