# V4 补齐试用接待与跟进话术记录

## 一句话结论

已补齐 `docs/40-执行/25-试用接待与跟进话术.md`，现在第一轮试用不再只会“发文案”和“记反馈”，也有了“有人回复后怎么接、怎么轻跟进”的固定话术。

## 动作

1. 新增现行标准件 `docs/40-执行/25-试用接待与跟进话术.md`。
2. 话术覆盖 8 个高频场景：
   - 第一次回复
   - 资格确认
   - 最短上手路径
   - 不会配置时怎么接
   - 24 小时轻跟进
   - 72 小时最后一次跟进
   - 卡住时怎么收信息
   - 顺利试完后怎么追问最有用
3. 同步更新 `docs/40-执行/README.md`、`docs/00-导航/02-现行标准件总览.md`、`docs/README.md` 与 `README.md`，把接待话术接入现行导航与首页入口。

## 理由

- 现在最大掉点不是“没人看到”，而是“有人回了以后接不住”。
- 如果回复后还要临时现编，第一轮转化率会被不必要地耗掉。
- 先把接待和跟进口径固定下来，后面才方便判断问题到底出在文案、上手，还是后续跟进。

## 结果

- 当前 Phase 1 公开试用链又往前补了一格：
  - 会发
  - 会记
  - 现在也会接
- 主公现在已经可以直接拿固定话术去接第一批回复，不必每次临场改口。
