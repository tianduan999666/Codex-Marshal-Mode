# 执行区证据稿归档规则

最后更新：2026-04-01
状态：现行归档规则
适用范围：`docs/40-执行/` 与 `docs/90-归档/`
说明：本文件用于说明哪些执行区证据稿应迁入归档，以及迁移后的维护方式

## 一句话结论

当执行区的时间戳文档已经完成其阶段使命，且已有固定编号标准件承接现行口径时，应迁入 `docs/90-归档/`。

## 适用对象

- 带时间戳的提炼稿、冻结稿、过程说明稿
- 已被固定编号标准件替代的过程性文档
- 仅保留作证据、复盘、追溯使用的文档

## 迁入条件

同时满足以下条件时，应考虑归档：

1. 已存在对应的现行标准件
2. 该文档不再承担当前对外唯一口径
3. 继续留在 `docs/40-执行/` 会增加阅读噪音
4. 迁入后不会影响现行路径的理解

## 固定节奏

为避免执行区长期堆积，归档候选至少按以下节奏复核：

1. 当本轮新增或改动了 `docs/40-执行/` 下的时间戳证据稿时，收口前复核一次是否满足迁入条件。
2. 当某份时间戳证据稿已被固定编号现行件承接后，不继续拖延到很多轮之后，应在当前阶段收口时复核一次是否归档。
3. 当准备做公开入口同步、公开提交或阶段收口时，若 `docs/40-执行/` 仍留有时间戳证据稿，应追加一次归档候选复核。
4. 若本轮复核后没有候选，不强行迁档；只需在任务包或本地日志中明确写明“本轮无待迁归档候选”。

## 不应归档的内容

- 仍在承担当前唯一口径的固定编号文件
- 仍在持续迭代中的现行标准件
- 当前轮正在直接使用的主文档
