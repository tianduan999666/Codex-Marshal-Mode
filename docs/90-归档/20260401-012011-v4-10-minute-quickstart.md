# V4 补齐 10 分钟上手指南记录

时间：2026-04-01 01:20:11

## 一句话结论

已补齐 `docs/40-执行/22-10分钟上手指南.md`，现在第一次来的 Windows 用户可以按“初始化 → 建任务 → 回 Codex 面板开工 → 收口 → 下次继续”五步跑通，不必先读整仓规则文档。

## 动作

1. 新增现行标准件 `docs/40-执行/22-10分钟上手指南.md`。
2. 指南只保留新手第一次真正需要的内容：
   - 初始化
   - 建第一个任务
   - 回官方 Codex 面板开工
   - 做完怎么结束
   - 下次怎么继续
3. 指南刻意避免把新手直接扔进一堆术语，只保留能直接照做的命令与话术。
4. 同步更新 `docs/40-执行/README.md`、`docs/00-导航/02-现行标准件总览.md`、`docs/README.md` 与 `README.md`，把新指南接入现行导航与公开入口。
5. 把首页“当前最该补齐”从“上手指南 + 公开试用文案”收敛为只剩“公开试用文案”。

## 理由

- 当前 Phase 1 最重要的不是继续堆维护能力，而是把已经做好的能力变成新手第一次就能照着走的路径。
- `一键初始化` 与 `一键新任务` 已经落地，如果没有一份真正给新手看的指南，首次体验仍会卡在“我下一步到底按什么顺序做”。
- 主公已明确默认入口是官方 Codex 面板，因此指南必须围绕“尽快回面板继续”来写，而不是把人长期留在终端里。

## 结果

- 现在公开入口已具备一条更完整的 Phase 1 路径：
  - `initialize-workspace.ps1`
  - `new-task.ps1`
  - `22-10分钟上手指南.md`
- 第一次来的用户，至少已经有机会在 10 分钟内跑通第一轮最小体验。
- 当前 Phase 1 剩余最重要公开缺口，只剩一段短小、能拿去发帖的试用文案。
