# docs 目录说明

普通用户安装与首次开工以仓根 `README.md` 为准；这里不再重复安装卡片和首轮传令，只保留文档导航与现行入口。默认安装不会静默覆盖你现有的 `~/.codex/config.toml`。

## 启动阶段入口

- `00-导航/01-V4-重启导读.md` 是启动阶段唯一对外总入口。
- 启动阶段核心入口以 `00-导航/01-V4-重启导读.md` 的 `先看什么` 为准，`docs/README.md` 不再重复抄整套入口清单。
- 启动阶段顺序以 `00-导航/01-V4-重启导读.md` 的 `启动阶段真源` 为准；需要细项时直接查看该文档。

## 继续深读

- 核心治理与公开边界以 `reference/01-反屎山AI研发执行总纲（Codex专用浓缩对照版）.md`、`reference/02-仓库卫生与命名规范.md`、`30-方案/02-V4-目录锁定清单.md`、`30-方案/08-V4-治理审计候选规范.md`、`40-执行/10-本地安全提交流程.md`、`40-执行/14-维护层动作矩阵与收口检查表.md`、`40-执行/21-关键配置来源与漂移复核模板.md` 为准。
- 执行区起步与试跑入口以 `40-执行/README.md` 的 `当前现行标准件` 为准；需要任务包规范、模板或面板入口验收时直接查看该文档。
- 需要冻结当前阶段“必须继续做 / 可以延后做 / 明确不做”的产品边界时，直接看 `40-执行/36-V4-丞相模式封顶边界清单.md`。
- 需要直接进入“闭源业务开发阶段怎么最短使用丞相”的打法时，看 `40-执行/37-V4-业务开发阶段最短作战手册.md`；只想一页记住高频面板传令时，看 `40-执行/38-V4-面板传令1页速查卡.md`；想看当前冻结下来的 5 个高频复用件时，看 `40-执行/39-V4-高频复用件总表.md`。
