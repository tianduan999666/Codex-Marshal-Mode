# 公开试用招募文案包

最后更新：2026-04-01
状态：现行标准件
适用范围：第一轮公开试用招募
说明：默认围绕官方 Codex 面板来讲，不再把主入口写回 Claude

## 一句话结论

如果现在要发第一轮试用招募，优先用下面 3 个版本：`V2EX`、`少数派`、`朋友圈`。先发出去，先拿回复，不要继续闷头补内部细节。

## 先记住这 3 条

1. 先讲结果，不先讲架构。
2. 先讲“下次不用重讲背景”，不先讲规则名词。
3. 先讲“官方 Codex 面板可继续工作”，不讲复杂维护细节。

## V2EX 版

适合：开发者、独立开发、效率工具讨论区

### 标题

```text
我做了一个让 Codex 会话能直接续上的小工具，想找几位 Windows 开发者试用
```

### 正文

```text
我最近把自己一直在用的一套继续工作工具整理成了公开试用版。

它主要解决一个很具体的问题：每次打开官方 Codex 面板开新会话，不想再把背景重讲一遍。

现在这套东西已经能做到：
1. 一条命令初始化
2. 一条命令创建新任务
3. 回到 Codex 面板后，直接接着做

目前先支持 Windows 开发者试用。
如果你平时会在 Codex 面板里持续做项目，愿意帮我试一轮，我可以免费帮你配第一套，也想听听你最真实的卡点。
```

## 少数派版

适合：效率、工作流、工具整理风格

### 标题

```text
我在做一套让 Codex 会话不再“失忆”的小工具，想邀请几位读者试用
```

### 正文

```text
这套工具想解决的不是“怎么多做一个按钮”，而是一个很烦的小问题：
每次打开官方 Codex 面板开新会话，常常还没开始干活，就得先重讲背景。

我现在把自己的这套继续工作流程整理成了公开试用版：
- 可以一键初始化
- 可以一键创建任务
- 然后直接回到 Codex 面板继续做

目前先支持 Windows 开发者。
如果你愿意试，我可以免费帮你完成第一轮配置，也欢迎直接告诉我：你最怕哪一步最麻烦。
```

## 朋友圈版

适合：熟人、轻量转介绍

### 正文

```text
我最近整理了一套给官方 Codex 面板用的继续工作小工具。
现在开新会话，不用再把背景重讲一遍，可以直接接着做。
目前先支持 Windows 开发者试用。
如果你想试，我可以免费帮你配第一套；如果你身边有人正好有这个痛点，也欢迎帮我转一下。
```

## 发出去前只检查这 4 项

1. 有没有把入口写成官方 Codex 面板。
2. 有没有先讲“能直接继续”，而不是先讲内部规则。
3. 有没有明确当前只先支持 Windows 开发者。
4. 有没有留下一个明确动作：`想试就回我`。

## 发出去后先记这 3 个数

1. 有多少人看见后回复。
2. 回复的人里，有多少人真的愿意试。
3. 真正试的人，第一卡点在哪一步。

## 下一步

1. 先选一个渠道发，不要三处同时铺太大。
2. 先收第一轮真实回复，再决定要不要继续改文案。
3. 若出现反复相同的卡点，再回仓里改对应入口或说明。
