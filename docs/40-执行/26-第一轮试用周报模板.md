# 第一轮试用周报模板

最后更新：2026-04-01
状态：现行标准件
适用范围：第一轮公开试用发出后的 7 天复盘
说明：目标不是写漂亮周报，而是用一页纸看清“渠道、文案、上手、回到 Codex 面板继续”四件事

## 一句话结论

第一轮试用发出去一周后，不要凭感觉决定下一刀。直接按这份模板复盘，先看哪里真正掉人，再决定改文案、改入口，还是改流程。

## 这一周先看 6 个事实

1. 一共发了几个渠道
2. 一共收到多少回复
3. 有多少人明确说愿意试
4. 有多少人真的开始试
5. 有多少人成功回到官方 Codex 面板继续
6. 最大卡点到底卡在初始化、建任务，还是回到面板继续

## 怎么用这份周报

1. 先从 `docs/40-执行/24-第一轮试用反馈记录表.md` 抄事实，不要凭印象写。
2. 再从 `docs/40-执行/25-试用接待与跟进话术.md` 回看，确认是不是接待动作本身在掉人。
3. 只挑出现次数最多的 1 到 2 个问题进入下一刀。
4. 周报写完后，立刻把下一步改动落成现行件或脚本，不要只停留在判断。

## 最后提醒

- 回复多，不等于需求成立；真正开始试，才算更强信号。
- 有人说好，不等于路径够轻；能顺利回到官方 Codex 面板继续，才算真的顺。
- 第一轮周报的价值，不是证明自己做得对，而是尽快找到最该改的一刀。
