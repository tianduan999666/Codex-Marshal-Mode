最后更新：2026-03-29

适用范围：当前仓维护层动作
说明：本文件用于把维护层常用动作统一收口，降低学习成本与误操作概率
补充：上方“当前维护层能力”区的文档列表，同时作为 `README.md` 与 `docs/README.md` 的维护层补充入口校验真源；“维护层主线真源”区作为维护层主线顺序校验真源

## 一句话结论

当需要做维护层动作时，先看本入口，再决定走哪条具体流程，而不是在多份文档之间来回找。

## 推荐使用顺序

1. 先判断自己是在“提交”“起包”“归档”“入口同步”“拍板准备”还是“治理复核”
2. 再进入对应的单项规则文档、动作矩阵或拍板规范
3. 当前轮若影响公开口径或现行标准件，收口前追加一次治理审计复核
4. 若当前轮还需要说明配置来源、版本依据或漂移检查，继续使用 `21` 追加落盘
5. 若当前轮准备推送公开改动，确认 `pre-push` 治理门禁已安装并可自动触发
6. 完成动作后，按收口检查表确认本轮已闭环
7. 最后再回到现行标准件总览确认入口是否需要同步

## 当前默认原则

- 维护层动作继续视为维护层，不对普通面板使用者外露复杂终端流程。
- 维护层动作优先复用当前仓已有规则与脚本，不另起外部依赖。
- 维护层动作完成后，如影响入口口径，应同步更新总览、首页或 docs 入口。
