# 维护层总入口

最后更新：2026-03-29
状态：现行维护入口
适用范围：当前仓维护层动作
说明：本文件用于把维护层常用动作统一收口，降低学习成本与误操作概率

## 一句话结论

当需要做维护层动作时，先看本入口，再决定走哪条具体流程，而不是在多份文档之间来回找。

## 当前维护层能力

### 1. 本地安全提交

- 文档：`docs/40-执行/10-本地安全提交流程.md`
- 适用：连续小步提交、避免 `.git/index.lock` 冲突

### 2. 任务包半自动起包

- 文档：`docs/40-执行/11-任务包半自动起包.md`
- 脚本：`.codex/chancellor/create-task-package.ps1`
- 适用：需要快速生成任务包 5 件套骨架

### 3. 执行区证据稿归档

- 文档：`docs/90-归档/01-执行区证据稿归档规则.md`
- 适用：需要把已失去现行职责的时间戳过程稿迁出执行区

## 推荐使用顺序

1. 先判断自己是在“提交”“起包”还是“归档”
2. 再进入对应的单项规则文档
3. 完成动作后，再回到现行标准件总览确认入口是否需要同步

## 当前默认原则

- 维护层动作继续视为维护层，不对普通面板使用者外露复杂终端流程。
- 维护层动作优先复用当前仓已有规则与脚本，不另起外部依赖。
- 维护层动作完成后，如影响入口口径，应同步更新总览、首页或 docs 入口。

## 什么时候优先看这份入口

- 不确定该先看哪份维护文档时
- 准备开始维护层动作前
- 需要交接维护动作给下一位执行者时

## 本文档的价值

- 把零散维护规则收成单一入口。
- 缩短维护层上手路径。
- 降低因找错入口而导致的误操作概率。
