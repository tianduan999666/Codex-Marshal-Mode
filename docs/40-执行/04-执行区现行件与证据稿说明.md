# 执行区现行件与证据稿说明

最后更新：2026-03-29
状态：现行标准件
适用范围：`docs/40-执行/`
说明：本文件用于区分执行区中哪些文件是当前口径，哪些只是过程证据，降低后续维护与检索成本

## 一句话结论

在 `docs/40-执行/` 下，固定编号文件是现行标准件，带时间戳的文件默认是证据稿或过程稿。

## 现行标准件

当前执行区以以下文件为准：

1. `01-任务包规范.md`
2. `02-任务包模板.md`
3. `03-面板入口验收.md`
4. `04-执行区现行件与证据稿说明.md`

## 证据稿与过程稿

以下类型默认不是现行标准件：

- 带时间戳的执行文档
- 某轮推进中的提炼稿、冻结稿、过程说明稿
- 仅用于还原当时判断过程的阶段性文档

## 使用顺序

1. 先读 `01-任务包规范.md`
2. 再读 `02-任务包模板.md`
3. 进入试跑前读 `03-面板入口验收.md`
4. 需要追溯历史判断时，再看时间戳证据稿

## 命名与维护规则

- 新增现行标准件时，优先使用固定编号文件名，再补本文件中的列表。
- 新增带时间戳的过程稿时，不得默认视为现行标准件。
- 若固定编号文件与时间戳稿出现差异，以固定编号文件为准。
- 若某份时间戳稿已失去参考价值，应转入 `docs/90-归档/` 而不是继续留在执行区长期混放。
