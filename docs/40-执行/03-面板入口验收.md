# 面板入口验收

最后更新：2026-03-29
状态：现行标准件
适用阶段：`V4-Trial`
说明：本验收单只覆盖官方面板入口与当前仓规则衔接，不扩展到真实任务试跑本身

## 一句话目标

确保主公从官方面板进入时，入口语义稳定、任务包衔接明确、改动后有固定人工验收闭环。

## 验收范围

- `丞相：` 首句入口风格
- 面板内置命令的最小响应口径
- 入口与 `.codex/chancellor/tasks/` 的衔接关系
- 入口相关改动后的人工复验步骤

## 本轮不验收

- 真实业务任务是否跑通
- 安装链与发布链是否稳定
- 多 Agent、长期记忆、独立 UI

## 两条维护命令边界

- `丞相修复`：在安全边界内尝试自动修复常见问题。
- `丞相验板`：给出进入官方面板人工验收的固定步骤。

## 固定人工验收步骤

1. 如有入口相关改动，先完成必要安装或同步动作。
2. 新开一个聊天会话，不复用旧会话。
3. 第一条消息输入：`丞相：测试入口是否稳态`。
4. 检查回复是否使用固定开头：`丞相亮启奏：谨呈本次事宜。`
5. 检查回复语气是否符合丞相模式，不回退为普通口吻。
6. 输入 `丞相状态`，检查是否能给出当前模式、状态与下一步的人话结论。
7. 若本地存在激活任务，检查回复口径是否与当前任务状态一致。

## 三条核心命令验收口径

- `丞相版本`：返回当前丞相模式版本与版本来源。
- `丞相检查`：做最小必要检查并返回人话结论。
- `丞相状态`：汇报当前模式、是否稳态、下一步。

## 通过标准

- `丞相：...` 首句能稳定进入丞相模式。
- 回复开头使用固定文案，且口吻不漂移。
- `丞相版本` 能返回当前丞相模式版本与版本来源。
- `丞相检查` 能做最小必要检查并返回人话结论。
- `丞相状态` 能汇报当前模式、是否稳态、下一步。
- 若存在激活任务，入口口径与本地任务状态不冲突。
- 入口相关改动后，能通过“新开会话 + 首句验板”完成复验。

## 失败信号

- 首句未进入丞相模式。
- 固定开头丢失或风格回退为普通模式。
- `丞相版本` 未返回当前丞相模式版本与版本来源。
- `丞相检查` 未做最小必要检查或未返回人话结论。
- `丞相状态` 未汇报当前模式、是否稳态、下一步。
- 入口回复与本地激活任务状态明显冲突。

## 失败后的处置动作

1. 先停止继续扩展任务范围。
2. 回看最近入口相关变更是否涉及 `AGENTS.md`、规则文档或安装同步动作。
3. 重新执行必要同步动作后，新开会话再次验板。
4. 若仍失败，记录为入口缺陷，不带着问题进入真实任务试跑。

## 与试跑阶段的关系

- 本文档是进入真实任务试跑前的固定门槛。
- 只有入口验收通过后，才进入后续真实任务闭环验证。
- 本文档可进入公开仓；真实运行态与日志继续只留本地。
