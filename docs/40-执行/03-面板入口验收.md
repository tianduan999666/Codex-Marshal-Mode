# 面板入口验收

最后更新：2026-04-02
状态：现行标准件
适用阶段：`V4-Target / 自用 MVP`
说明：本验收单只覆盖官方面板入口与当前仓规则衔接，不扩展到真实任务试跑本身
补充：本文中的 `验真`、`修复`、`检查` 只针对丞相运行环境与真源，不检查你的项目代码

## 一句话目标

确保主公从官方面板进入时，入口语义稳定、任务包衔接明确、改动后有固定人工验收闭环。

## 验收范围

- `传令：` 首句入口风格
- 面板内置命令的最小响应口径
- 入口与 `.codex/chancellor/tasks/` 的衔接关系
- 入口相关改动后的人工复验步骤

## 验收范围固定槽位

- `首句入口风格`：`传令：` 首句入口风格。
- `最小响应口径`：面板内置命令的最小响应口径。
- `任务包衔接`：入口与 `.codex/chancellor/tasks/` 的衔接关系。
- `人工复验步骤`：入口相关改动后的人工复验步骤。

## 本轮不验收

- 真实业务任务是否跑通
- 安装链与发布链是否稳定
- 多 Agent、长期记忆、独立 UI

## 帮助命令公开用法

- `传令 帮助`：显示当前用法、命令与注意事项。

## 帮助命令固定结构

- `当前用法`：先说明当前默认入口与适用场景。
- `面板命令`：再列出当前面板传令命令及其含义。
- `注意事项`：最后提示维护层动作、安全边界与人工验板提醒。

## 当前用法固定子项

- `默认入口`：当前默认用户入口是官方 Codex 面板。
- `适用场景`：普通用户日常使用优先在面板内完成，终端仅保留给维护、安装、迁移、排障。
- `维护层说明`：如必须落到脚本层，要明确说明“这是维护层动作”。

## 面板命令固定子项

- `命令顺序`：按传令 帮助、传令 状态、传令 检查、传令 修复、传令 验板、传令 版本的顺序展示。
- `命令含义`：每条命令都要带一句人话含义，不得只列命令名。
- `公开边界`：普通用户只暴露面板命令，不主动推荐终端丞相别名。

## 注意事项固定子项

- `安全边界`：普通用户入口优先留在官方面板，维护层动作不向普通用户外露复杂终端流程。
- `维护层动作`：如必须落到脚本层，要明确说明“这是维护层动作”。
- `新开会话验板提醒`：入口相关改动后，建议新开官方面板会话做人眼验板。

## 两条维护命令边界

- `传令 修复`：在安全边界内尝试自动修复常见问题。
- `传令 验板`：给出进入官方面板人工验收的固定步骤。

## 传令 修复固定槽位

- `修复范围`：只在安全边界内处理丞相运行环境的当前常见问题，不改你的项目代码。
- `处理方式`：优先尝试自动修复当前已知常见问题。
- `升级条件`：超出安全边界或无法自动修复时停止扩展并提示人工处理。

## 传令 验板固定槽位

- `触发场景`：入口相关改动后或切换后进入官方面板做人眼验板。
- `验板动作`：给出进入官方面板人工验收的固定步骤。
- `验板目标`：确认版本、模式与入口表现是否稳态。

## 固定人工验收步骤

1. 如有入口相关改动，先完成必要安装或同步动作。
2. 新开一个聊天会话，不复用旧会话。
3. 第一条消息输入：`传令：测试入口是否稳态`。
4. 检查回复是否使用固定开头：`丞相亮启奏：谨呈本次事宜。`
5. 检查回复语气是否符合丞相模式，不回退为普通口吻。
6. 输入 `传令 状态`，检查是否能给出当前模式、状态与下一步的人话结论。
7. 若本地存在激活任务，检查回复口径是否与当前任务状态一致。

## 固定人工验收步骤固定子项

- `预处理动作`：如有入口相关改动，先完成必要安装或同步动作。
- `新开会话`：新开一个聊天会话，不复用旧会话。
- `首句验板`：第一条消息输入：`传令：测试入口是否稳态`。
- `开头校验`：检查回复是否使用固定开头：`丞相亮启奏：谨呈本次事宜。`
- `语气校验`：检查回复语气是否符合丞相模式，不回退为普通口吻。
- `状态校验`：输入 `传令 状态`，检查是否能给出当前模式、状态与下一步的人话结论。
- `任务一致性`：若本地存在激活任务，检查回复口径是否与当前任务状态一致。

## 三条核心命令验收口径

- `传令 版本`：返回当前版本与版本来源。
- `传令 检查`：做最小必要检查并返回人话结论。
- `传令 状态`：汇报当前模式、是否稳态、下一步。

## 传令 版本固定槽位

- `版本号`：返回当前版本号。
- `版本来源`：明确当前版本来自哪个版本来源。
- `真源路径`：优先说明当前版本真源路径为 `codex-home-export/VERSION.json`。

## 传令 检查固定槽位

- `检查范围`：只检查丞相运行环境、版本、真源与关键文件，不检查你的项目代码。
- `检查结论`：用人话汇报当前是否存在明显异常或风险。
- `建议动作`：如发现问题，给出下一步建议或引导到修复 / 验板。

## 传令 状态固定槽位

- `当前模式`：明确当前处于丞相还是都督。
- `稳态判断`：明确当前是否稳态。
- `下一步`：给出最小下一步建议。

## 通过标准

- `传令：...` 首句能稳定进入丞相模式。
- 回复开头使用固定文案，且口吻不漂移。
- `传令 帮助` 能显示当前用法、命令与注意事项。
- `传令 帮助` 能按“当前用法 → 面板命令 → 注意事项”组织输出。
- `传令 帮助` 的当前用法能覆盖“默认入口 → 适用场景 → 维护层说明”。
- `传令 帮助` 的面板命令说明能覆盖“命令顺序 → 命令含义 → 公开边界”。
- `传令 帮助` 的注意事项能覆盖“安全边界 → 维护层动作 → 新开会话验板提醒”。
- `固定人工验收步骤` 能覆盖“预处理动作 → 新开会话 → 首句验板 → 开头校验 → 语气校验 → 状态校验 → 任务一致性”。
- `传令 版本` 能返回当前版本与版本来源。
- `传令 版本` 的输出能覆盖“版本号 → 版本来源 → 真源路径”。
- `传令 检查` 能做最小必要检查并返回人话结论。
- `传令 检查` 的输出能覆盖“检查范围 → 检查结论 → 建议动作”。
- `传令 状态` 能汇报当前模式、是否稳态、下一步。
- `传令 状态` 的输出能覆盖“当前模式 → 稳态判断 → 下一步”。
- `传令 修复` 的边界说明能覆盖“修复范围 → 处理方式 → 升级条件”。
- `传令 验板` 的边界说明能覆盖“触发场景 → 验板动作 → 验板目标”。
- 若存在激活任务，入口口径与本地任务状态不冲突。
- 入口相关改动后，能通过“新开会话 + 首句验板”完成复验。

## 通过标准固定子项

- `模式稳定`：首句进入丞相模式，固定开头不丢失，语气不漂移。
- `帮助完整`：帮助输出覆盖结构与子项模板。
- `命令口径`：版本、检查、状态与维护边界口径完整。
- `任务一致`：若存在激活任务，入口口径与本地任务状态不冲突。
- `复验闭环`：入口相关改动后，可通过新开会话与首句验板完成复验。

- 固定开头丢失或风格回退为普通模式。
- `传令 帮助` 未显示当前用法、命令与注意事项。
- `传令 帮助` 未按“当前用法 → 面板命令 → 注意事项”组织输出。

## 失败信号固定子项

- `模式失稳`：首句未进入丞相模式，或固定开头丢失、语气回退。
- `帮助缺项`：帮助输出未覆盖固定结构或固定子项模板。
- `命令漂移`：版本、检查、状态或维护边界口径出现缺失或漂移。
- `任务冲突`：入口回复与本地激活任务状态明显冲突。
- `复验失败`：重新执行必要同步动作后，仍无法通过新开会话与首句验板完成复验。

## 失败后的处置动作

1. 先停止继续扩展任务范围。
2. 回看最近入口相关变更是否涉及 `AGENTS.md`、规则文档或安装同步动作。
3. 重新执行必要同步动作后，新开会话再次验板。
4. 若仍失败，记录为入口缺陷，不带着问题进入真实任务试跑。

## 失败后的处置动作固定子项

- `停止扩展`：先停止继续扩展任务范围。
- `回看变更`：回看最近入口相关变更是否涉及 `AGENTS.md`、规则文档或安装同步动作。
- `重新验板`：重新执行必要同步动作后，新开会话再次验板。
- `缺陷收口`：若仍失败，记录为入口缺陷，不带着问题进入真实任务试跑。

- 本文档可进入公开仓；真实运行态与日志继续只留本地。
