# 面板入口验收

最后更新：2026-04-05
状态：现行标准件
适用阶段：`V4-Target / 自用 MVP`
说明：本验收单只覆盖官方面板入口与当前仓规则衔接，不扩展到真实任务试跑本身
补充：`提示：丞相在检查阶段只检查自己，不会查看你的项目；执行阶段只按你的传令办事，不会擅自审查项目。`

## 一句话目标

确保主公新开对话时，一眼就懂怎么用；入口不歧义，开场白固定，查看口径稳定，任务入口可以直接开工。

## 验收范围

- 新对话自动提示是否清楚
- `传令：XXXX` 是否能作为唯一做事入口
- `传令：状态 / 传令：版本 / 传令：升级` 是否口径稳定
- 固定开场白、检查阶段边界提示与任务衔接是否稳定

## 固定公开协议

- 唯一做事入口：`传令：XXXX`
- 3 个可查命令：`传令：状态 / 传令：版本 / 传令：升级`
- 新对话自动提示：`例如：传令：计算1+1=?`
- 默认开场白：`🪶 军令入帐。亮，即刻接管全局。`
- 固定开工骨架：`开场白 → 接令句`
- 进入检查阶段时：`开场白 → 固定边界提示 → 接令句`
- 对外流程：`先确认丞相能正常接到传令 → 再确认丞相自身状态良好 → 接着把丞相调整到最佳工作状态 → 丞相记录这次要做的任务 → 丞相开始执行任务`
- 固定边界提示：`提示：丞相在检查阶段只检查自己，不会查看你的项目；执行阶段只按你的传令办事，不会擅自审查项目。`
- 固定收口格式：`已完成 / 结果 / 下一步`
- 固定渲染真源：`codex-home-export/VERSION.json` 与 `codex-home-export/render-panel-response.ps1`
- 固定路由入口：`codex-home-export/invoke-panel-command.ps1`
- 固定验板入口：`codex-home-export/start-panel-acceptance.ps1` 取预期口径时，也必须走 `invoke-panel-command.ps1`
- 固定状态栏顺序：以 `status_bar_slots` 为准，不允许自行换序

## 本轮不验收

- 真实业务任务是否跑通
- 安装链与发布链是否稳定
- 多 Agent、长期记忆、独立 UI

## 三条可查命令口径

- `传令：版本`：返回当前版本与真源。
- `传令：状态`：汇报当前状态与下一步。
- `传令：升级`：只在用户主动提出时，再处理升级动作。

## 传令：版本固定槽位

- `版本号`：返回当前版本号。
- `版本来源`：明确当前版本来自哪个版本来源。
- `真源路径`：优先说明当前版本真源路径为 `codex-home-export/VERSION.json`。

## 传令：状态固定槽位

- 固定顺序来源：`codex-home-export/VERSION.json` 的 `status_bar_slots`；验收时不接受临时调序。
- `版本`：当前版本号。
- `上次检查`：最近一次检查时间或结论。
- `自动修复`：是否触发过自动修复。
- `关键文件一致性`：关键文件是否一致。
- `当前模式`：当前处于丞相还是维护。
- `当前任务`：当前是否存在激活任务。

## 最小过程提示固定句

- `接令`：`军令已明，亮先接手。`
- `研判`：`亮先看清症结，再动手。`
- `拆解`：`此事可拆，亮按最短路径推进。`
- `调度`：`所需动作已排定，开始推进。`
- `收束`：`主干已稳，亮正在收束余项。`
- `收口`：`此事已交卷，现呈结果。`
- `显示规则`：过程提示一次只显示 1 句，不随机刷屏。

## 传令：升级固定槽位

- `触发方式`：只在用户主动输入 `传令：升级` 时触发。
- `处理边界`：只处理丞相自身升级或同步，不擅自升级用户项目。
- `默认策略`：未收到明确升级传令时，不自动升级。

## 固定人工验收步骤

1. 如有入口相关改动，先完成必要安装或同步动作。
2. 新开一个聊天会话，不复用旧会话。
3. 先看系统是否给出示例：`例如：传令：计算1+1=?`
4. 第一条消息输入：`传令：测试入口是否稳态`
5. 检查回复是否使用固定开场白：`🪶 军令入帐。亮，即刻接管全局。`
6. 检查回复是否只在进入检查阶段时带出固定边界提示；若未进入检查阶段，不重复显示，也不把检查对象误说成你的项目。
7. 输入 `传令：版本`，检查版本号、版本来源、真源路径是否清楚。
8. 输入 `传令：状态`，检查是否能按固定 6 行说清当前状态。
9. 如需确认升级口径，再输入 `传令：升级`，检查是否明确“默认不自动升级，需用户主动提出”。
10. 若本地存在激活任务，检查回复口径是否与当前任务状态一致。

## 通过标准

- 新对话自动提示清楚，用户能直接照抄。
- `传令：...` 首句能稳定进入丞相模式。
- 开场白固定为 `🪶 军令入帐。亮，即刻接管全局。`
- 开工默认骨架稳定，不回退为散乱长段落。
- 仅在进入检查阶段时显示固定边界提示；不检查时不重复显示。
- 边界提示口径稳定，不回退为“检查项目”。
- `传令：版本` 能说清版本号、版本来源、真源路径。
- `传令：状态` 能按固定 6 行说清当前状态。
- 过程提示一次只显示 1 句，不刷屏。
- 若触发收口，能按 `已完成 / 结果 / 下一步` 收束。
- `传令：升级` 能明确“用户主动提出才处理，不自动升级”。
- 若存在激活任务，入口口径与本地任务状态不冲突。
- 入口相关改动后，能通过“新开会话 + 首句验板”完成复验。

## 失败信号

- 新对话没有给出示例，用户仍要猜怎么开口。
- 首句未进入丞相模式。
- 开场白不是 `🪶 军令入帐。亮，即刻接管全局。`
- 开工骨架顺序混乱，或一上来就刷长段文案。
- 没进入检查阶段却重复显示边界提示，或该显示时缺失。
- 边界提示把检查对象说成了用户项目。
- `传令：版本` 未说清版本号、版本来源或真源路径。
- `传令：状态` 不能按固定 6 行说清当前状态。
- 过程提示一次出现多句，或明显随机刷屏。
- 收口没有按 `已完成 / 结果 / 下一步` 组织。
- `传令：升级` 没有说清“默认不自动升级”。
- 入口回复与本地激活任务状态明显冲突。

## 失败后的处置动作

1. 先停止继续扩展任务范围。
2. 回看最近入口相关变更是否涉及 `AGENTS.md`、规则文档或安装同步动作。
3. 重新执行必要同步动作后，新开会话再次验板。
4. 若仍失败，记录为入口缺陷，不带着问题进入真实任务试跑。

## 与试跑阶段的关系

- 本文档是进入真实任务试跑前的固定门槛。
- 只有入口验收通过后，才进入后续真实任务闭环验证。
- 本文档可进入公开仓；真实运行态与日志继续只留本地。
