# 10 分钟上手指南

最后更新：2026-04-05
状态：现行标准件
适用范围：第一次试用丞相模式的 Windows 用户
说明：日常入口优先在官方 Codex 面板；终端只用来做准备动作

## 一句话结论

你第一次用，先做一次安装。做完以后，日常开工优先回官方 Codex 面板直接说：`传令：修一下登录页`。系统对外会按“先确认丞相能正常接到传令 → 再确认丞相自身状态良好 → 接着把丞相调整到最佳工作状态 → 丞相记录这次要做的任务 → 丞相开始执行任务”解释流程；内部仍会做最小必要的检查、验真与修复，然后自动建任务并留在当前会话继续。只有真的进入检查阶段时，才会额外显示那句固定边界提示。

## 先准备什么

- 一台 Windows 电脑
- 已安装 Git
- 已能打开官方 Codex 面板
- 已把 `Codex-Chancellor-Mode` 仓放到本机目录里

## 第 0 步（1 分钟，换电脑时先做）：拉仓

```powershell
git clone https://github.com/tianduan999666/Codex-Chancellor-Mode.git
cd Codex-Chancellor-Mode
```

## 日常最短开工（30 秒）

新对话里如果你一时不知道怎么开口，先照着这句发：

```text
例如：传令：计算1+1=?
```

如果你已经完成过一次初始化，之后日常直接回官方 Codex 面板说：

```text
传令：修一下登录页
```

正常情况下，系统会自动做 5 件事：

- 先确认丞相能正常接到传令
- 再确认丞相自身状态良好
- 接着把丞相调整到最佳工作状态
- 然后记录这次要做的任务
- 最后直接开始执行

只有一句话入口失败时，你才需要回维护层手动跑脚本。

## 空文件夹快验（1 分钟）

如果你只是想判断“现在是不是新版本，入口正不正常”，可以在一个空文件夹里直接做这 4 步。空文件夹不会影响判断，因为检查阶段只检查丞相自己，不会先查看你的项目。

| 步骤 | 你输入什么 | 预期正常结果 |
| --- | --- | --- |
| 1 | 新开一个聊天会话 | 先看到示例：`例如：传令：计算1+1=?` |
| 2 | `传令：版本` | 能说清 `版本号 / 版本来源 / 真源路径` |
| 3 | `传令：状态` | 能按 6 行说清：`版本 / 上次检查 / 自动修复 / 关键文件一致性 / 当前模式 / 当前任务` |
| 4 | `传令：修一下登录页` | 先用固定开场白 `🪶 军令入帐。亮，即刻接管全局。`；若本轮真的进入检查阶段，再补固定边界提示，不会先跑去检查你的空文件夹项目 |

### 需要精确验收时

这份指南只保留快验标准，不再重复抄固定回复示例。
如果你要逐字核对 `传令：版本 / 传令：状态 / 传令：修一下登录页` 的预期口径，直接看 `03-面板入口验收.md`。

如果出现下面任一情况，直接判失败：

- 还在把旧入口词当正式入口
- 没说清版本来源或真源路径
- `传令：状态` 说不出 6 行固定状态栏
- 先去讨论你的空文件夹项目，而不是先按丞相自己的检查边界办事

如果你在维护层还想先看“真源现在会怎么说”，可直接运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\render-panel-response.ps1 -Kind hint
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\render-panel-response.ps1 -Kind version
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\render-panel-response.ps1 -Kind status
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\invoke-panel-command.ps1 "传令：状态"
```

这 3 条不是日常入口，只是维护层用来确认当前真源口径是否正常。

## 安装后要不要完全重启

最短判断：

- 先试 `换新对话`
- 还看到旧口径或旧鉴权，再 `重开 VSCode / 官方 Codex 面板`
- 一般不用 `重启电脑`

具体判断按这个表：

| 情况 | 你怎么做 | 原因 |
| --- | --- | --- |
| 刚执行完 `.\install.cmd` 或 `%USERPROFILE%\.codex\upgrade.cmd` | 先新开一个官方 Codex 面板对话 | 新对话会重新走当前入口链 |
| 新对话里已经能返回 `传令：版本` 的 3 行固定口径 | 不用再重启 | 说明新版本已经接管 |
| 新对话里仍是旧示例、旧开场白、旧入口词，或你刚改了全局 `provider / auth` 仍报旧鉴权 | 重开一次 VSCode / 官方 Codex 面板，再重试 | 多半是会话缓存还没刷新，新会话才会吃到最新入口与配置 |
| 只是丞相模式脚本升级，没有动系统环境 | 不用重启电脑 | 电脑重启通常没有额外收益 |

## 第 1 步（2 分钟，第一次才需要）：安装

在仓库根目录打开 PowerShell，运行：

```powershell
.\install.cmd
```

普通用户不要直接跑 `install-to-home.ps1`、`verify-cutover.ps1` 这类底层脚本；先走 `.cmd` 包装入口，能避开 PowerShell 5.1 的执行细节和编码摩擦。

这条命令默认只做普通用户需要记住的 3 件事：

- 同步最小运行文件，并把 `upgrade.cmd / self-check.cmd / rollback.cmd` 放到 `~/.codex`
- 自动做传令冒烟、真实鉴权探针；如果你已经登录过 Codex，再顺手做一次完整验真
- 默认保留你现有的 `~/.codex/config.toml`，不静默切换 provider / key

更细的受管文件与安装细节，统一看 `codex-home-export/README.md`。

如果你确实要显式套用仓内模板配置，再单独执行：

```powershell
.\install.cmd -ApplyTemplateConfig
```

如果你已经登录过 Codex，通常可以一路跑完；如果还没登录，脚本会告诉你下一步，不会直接把你卡死。

## 第 2 步（2 分钟，只有一句话入口失败时才需要）：手动建任务

如果你在面板里说了 `传令：修一下登录页`，但自动开工失败，这时再回维护层运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\new-task.ps1 -Title "修一下登录页"
```

运行后你会看到：

- 新任务编号
- 新任务目录
- 当前标题

当前默认会创建 `v4-target-*` 任务；如需补建 Trial 任务，再显式加 `-TaskNamespace trial`。
不用手工建文件，不用自己编号。

## 第 3 步（1 分钟）：回到官方 Codex 面板一句话开工

打开官方 Codex 面板，直接说：

```text
传令：修一下登录页
```

预期正常情况：

- 系统先自动检查本机生产态
- 如果发现可修复漂移，会先自动修一次
- 然后自动建任务
- 最后直接留在当前会话继续干活

这样做的目的只有一个：你只说一句话，其他让系统先做。

如果你只是想查看，不是想开工，记住这 3 句就够了：

```text
传令：状态
传令：版本
传令：升级
```

其中 `传令：升级` 只有你主动说了才会处理；系统默认不自动升级。

## 第 4 步（2 分钟）：任务做完怎么结束

当你觉得这一刀已经做完，就在官方 Codex 面板里说：

```text
传令：这一刀已完成，请收口。
请按 已完成 / 结果 / 下一步 的顺序整理，并把需要留痕的内容落盘。
如果这轮改了正式文件，请继续完成提交与推送。
```

你不需要自己手写一堆总结；先让 Codex 帮你收口。

## 第 5 步（1 分钟）：下次怎么继续

下次再打开官方 Codex 面板时，直接说：

```text
传令：继续当前任务
```

如果 `active-task.txt` 里已经有当前任务，系统应优先续上，不重复创建新任务。
如果你中途忘了自己在做什么，也可以先看当前激活任务，再回来继续。

## 最少只记住这 4 条

1. 日常入口在官方 Codex 面板，直接说：`传令：修一下登录页`。
2. 想查看时，只用 `传令：状态 / 传令：版本 / 传令：升级`。
3. 终端维护层只记 4 个动作：`install.cmd / upgrade.cmd / self-check.cmd / rollback.cmd`。
4. 做完一刀就让 Codex 收口，不要留半拉子现场。

## 常用命令（只给维护层备用）

```powershell
# 第一次安装
.\install.cmd

# 从任何目录升级
%USERPROFILE%\.codex\upgrade.cmd

# 从任何目录自检

# 从任何目录回滚
%USERPROFILE%\.codex\rollback.cmd
```

需要更深维护动作时，不在这份上手指南里继续展开，直接看 `docs/40-执行/13-维护层总入口.md` 与 `codex-home-export/README.md`。
