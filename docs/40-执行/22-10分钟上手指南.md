# 10 分钟上手指南

最后更新：2026-04-02
状态：现行标准件
适用范围：第一次试用丞相模式的 Windows 用户
说明：日常入口优先在官方 Codex 面板；终端只用来做准备动作

## 一句话结论

你第一次用，先做一次初始化。做完以后，日常开工优先回官方 Codex 面板直接说：`传令：我要做 XX`。系统对外会按“先确认丞相能正常接到传令 → 再确认丞相自身状态良好 → 接着把丞相调整到最佳工作状态 → 丞相记录这次要做的任务 → 丞相开始执行任务”解释流程；内部仍会做最小必要的检查、验真与修复，然后自动建任务并留在当前会话继续。

## 先准备什么

- 一台 Windows 电脑
- 已安装 Git
- 已能打开官方 Codex 面板
- 已把本仓放到本机目录里

## 日常最短开工（30 秒）

新对话里如果你一时不知道怎么开口，先照着这句发：

```text
例如：传令：计算1+1=?
```

如果你已经完成过一次初始化，之后日常直接回官方 Codex 面板说：

```text
传令：我要做 修复登录页 bug
```

正常情况下，系统会自动做 5 件事：

- 先确认丞相能正常接到传令
- 再确认丞相自身状态良好
- 接着把丞相调整到最佳工作状态
- 然后记录这次要做的任务
- 最后直接开始执行

只有一句话入口失败时，你才需要回维护层手动跑脚本。

## 第 1 步（2 分钟，第一次才需要）：初始化

在仓库根目录打开 PowerShell，运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\initialize-workspace.ps1
```

这条命令现在默认只做最小主链：

- 同步本机需要的最小运行文件
- 如果你已经登录过 Codex，就顺手做一次验真
- 明确告诉你下一步该建任务还是单独验真

它默认不再安装治理门禁，也不再自动创建示例任务。
如果你已经登录过 Codex，通常可以一路跑完；如果还没登录，脚本会告诉你下一步，不会直接把你卡死。

## 第 2 步（2 分钟，只有一句话入口失败时才需要）：手动建任务

如果你在面板里说了 `传令：我要做 XX`，但自动开工失败，这时再回维护层运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\new-task.ps1 -Title "修复登录页 bug"
```

运行后你会看到：

- 新任务编号
- 新任务目录
- 当前标题

当前默认会创建 `v4-target-*` 任务；如需补建 Trial 任务，再显式加 `-TaskNamespace trial`。
不用手工建文件，不用自己编号。

## 第 3 步（1 分钟）：回到官方 Codex 面板一句话开工

打开官方 Codex 面板，直接说：

```text
传令：我要做 修复登录页 bug
```

预期正常情况：

- 系统先自动检查本机生产态
- 如果发现可修复漂移，会先自动修一次
- 然后自动建任务
- 最后直接留在当前会话继续干活

这样做的目的只有一个：你只说一句话，其他让系统先做。

如果你只是想查看，不是想开工，记住这 3 句就够了：

```text
传令：状态
传令：版本
传令：升级
```

其中 `传令：升级` 只有你主动说了才会处理；系统默认不自动升级。

## 第 4 步（2 分钟）：任务做完怎么结束

当你觉得这一刀已经做完，就在官方 Codex 面板里说：

```text
传令：这一刀已完成，请收口。
请按 已完成 / 结果 / 下一步 的顺序整理，并把需要留痕的内容落盘。
如果这轮改了正式文件，请继续完成提交与推送。
```

你不需要自己手写一堆总结；先让 Codex 帮你收口。

## 第 5 步（1 分钟）：下次怎么继续

下次再打开官方 Codex 面板时，直接说：

```text
传令：继续当前任务
```

如果 `active-task.txt` 里已经有当前任务，系统应优先续上，不重复创建新任务。
如果你中途忘了自己在做什么，也可以先看当前激活任务，再回来继续。

## 最少只记住这 4 条

1. 日常入口在官方 Codex 面板，直接说：`传令：我要做 XX`。
2. 想查看时，只用 `传令：状态 / 传令：版本 / 传令：升级`。
3. 终端只做准备和兜底，不再是你每天开工的主入口。
4. 做完一刀就让 Codex 收口，不要留半拉子现场。

## 常用命令（只给维护层备用）

```powershell

```
