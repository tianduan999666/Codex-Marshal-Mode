# 10 分钟上手指南

最后更新：2026-04-01
状态：现行标准件
适用范围：第一次试用大都督模式的 Windows 用户
说明：日常入口优先在官方 Codex 面板；终端只用来做准备动作

## 一句话结论

你只需要做两件维护动作：先初始化，再建任务。之后就回到官方 Codex 面板继续，不用每次重新解释背景。

## 先准备什么

- 一台 Windows 电脑
- 已安装 Git
- 已能打开官方 Codex 面板
- 已把本仓放到本机目录里

## 第 1 步（2 分钟）：初始化

在仓库根目录打开 PowerShell，运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\initialize-workspace.ps1
```

这条命令会尽量自动帮你做完：

- 准备本机需要的最小文件
- 安装提交保护
- 创建第一个示例任务
- 条件满足时自动做一次检查

如果你已经登录过 Codex，通常可以一路跑完；如果还没登录，脚本会告诉你下一步，不会直接把你卡死。

## 第 2 步（2 分钟）：创建你的第一个任务

如果你不想用示例任务，直接运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\new-task.ps1 -Title "修复登录页 bug"
```

运行后你会看到：

- 新任务编号
- 新任务目录
- 一段可以直接贴到官方 Codex 面板的话

不用手工建文件，不用自己编号。

## 第 3 步（3 分钟）：回到官方 Codex 面板开工

打开官方 Codex 面板，新开一个会话，把脚本刚才打印的那段话直接贴进去。

如果你想手动输入，可以用这段：

```text
丞相：当前任务已创建。
任务编号：<把脚本输出的任务编号贴这里>
任务标题：<把脚本输出的任务标题贴这里>
请先判断瓶颈，给出最小可验证推进点，然后直接开始。
```

这样做的目的只有一个：让 Codex 直接接着干，不要先花 10 分钟听你重讲背景。

## 第 4 步（2 分钟）：任务做完怎么结束

当你觉得这一刀已经做完，就在官方 Codex 面板里说：

```text
丞相：这一刀已完成，请收口。
请按 已完成 / 结果 / 下一步 的顺序整理，并把需要留痕的内容落盘。
如果这轮改了正式文件，请继续完成提交与推送。
```

你不需要自己手写一堆总结；先让 Codex 帮你收口。

## 第 5 步（1 分钟）：下次怎么继续

下次再打开官方 Codex 面板时，直接说：

```text
丞相：继续当前任务，不用重新解释背景。
请先用三句话说明：当前任务、已完成、下一步。
```

如果你中途忘了自己在做什么，也可以先看脚本创建任务时写入的当前任务，再回来继续。

## 最少只记住这 3 条

1. 日常入口在官方 Codex 面板，不在终端。
2. 终端只做两件事：初始化、建任务。
3. 做完一刀就让 Codex 收口，不要留半拉子现场。

## 常用命令

```powershell
# 一键初始化
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\initialize-workspace.ps1

# 新建任务
powershell.exe -ExecutionPolicy Bypass -File .\codex-home-export\new-task.ps1 -Title "你的任务标题"
```
