# 朋友圈与熟人转介绍首发执行包

最后更新：2026-04-01
状态：现行标准件
适用范围：第一轮公开试用的朋友圈首发与熟人转介绍
说明：目标不是写成长文，而是给主公一份今天就能直接发的短文案、私聊跟进话术与 24 小时动作卡

## 一句话结论

如果第一轮改成先发 `朋友圈` 或先走熟人转介绍，就不要临场编。直接用这一版短文案和私聊话术发出去；发完后只按固定动作记反馈、接回复、做 24 小时判断。

## 先记住这 4 条

1. 朋友圈首发求的是熟人信任，不求长篇铺陈。
2. 首帖先讲结果，不先讲内部体系。
3. 主入口始终写成官方 Codex 面板，不写回 Claude。
4. 帖子发出后，先收真实回复，不先急着改文案。

## 今天直接发这一版

### 朋友圈正文

```text
我最近整理了一套给官方 Codex 面板用的继续工作小工具。
现在开新会话，不用再把背景重讲一遍，可以直接接着做。
目前先支持 Windows 开发者试用。
如果你想试，我可以免费帮你配第一套；如果你身边有人正好有这个痛点，也欢迎帮我转一下。
```

### 发给熟人的转介绍短话术

```text
我最近在整理一套给官方 Codex 面板用的继续工作小工具，
主要解决“每次开新会话都要重讲背景”这个问题。
现在先收 Windows 开发者做第一轮试用。
如果你身边刚好有人在 Codex 面板里持续做项目，方便的话帮我转一下；
如果对方愿意试，我可以直接带他走第一轮。
```

## 发出去前最后核对

- 有没有把入口写成官方 Codex 面板
- 有没有保留“Windows 开发者”这个筛选条件
- 有没有留下两个动作：`想试就回我`、`合适的话帮我转一下`
- 有没有避免把朋友圈写成一大段技术说明

## 发帖后 30 分钟动作卡

1. 把发帖时间、渠道、使用文案记进 `docs/40-执行/24-第一轮试用反馈记录表.md`
2. 把准备用的跟进话术对照 `docs/40-执行/25-试用接待与跟进话术.md` 再看一遍
3. 先不急着重写文案，先等第一批真实回复或转介绍

## 回复分流卡

### 对方说“我想试”

直接回：

```text
好呀，我这套现在先支持 Windows 开发者试用。
你如果方便，我先把仓库和最短上手路径发你：
1. 先跑一条初始化命令
2. 再建一个任务
3. 然后回官方 Codex 面板直接继续

你试的时候如果卡住，直接把报错或卡点发我就行，我会按第一轮试用帮你一起走通。
```

### 对方说“我先帮你转给朋友”

直接回：

```text
太好了，谢谢你。
你就把我刚那段话直接转给他就行；
如果他愿意试，让他直接来找我，我会把第一轮最短路径发给他。
```

### 对方说“我先看看，不一定试”

直接回：

```text
完全没关系，这一轮我主要是想先收第一手真实卡点。
你后面如果想试，直接回我一句就行；
我这边会继续把第一次上手路径压短。
```

## 发帖后 24 小时动作卡

### 先看这 4 个事实

- 有没有人回复
- 有没有人明确说愿意试
- 有没有人帮忙转介绍
- 最大卡点是在“没人接话”还是“有人接了但没继续”

### 然后只做这 3 件事

1. 用 `25-试用接待与跟进话术.md` 轻跟进一次
2. 把真实原话、人数和转介绍情况补进 `24-第一轮试用反馈记录表.md`
3. 不因为熟人礼貌性点赞或鼓励，就判断需求已经成立

## 这一刀之后接哪一份

1. 发帖当天先看 `docs/40-执行/27-第一轮真实发帖执行清单.md`
2. 如果第一站先发朋友圈或先走熟人转介绍，就继续看本页
3. 有回复后看 `docs/40-执行/25-试用接待与跟进话术.md`
4. 需要沉淀事实时看 `docs/40-执行/24-第一轮试用反馈记录表.md`
5. 满 7 天后看 `docs/40-执行/26-第一轮试用周报模板.md`

## 最后提醒

- 朋友圈更容易拿到熟人反馈，但也更容易混入礼貌性支持，要把“愿意试”与“只是点赞”分开记。
- 如果已经有人愿意帮转介绍，就先把人接住，这比继续补新文档更值钱。
- 这条渠道适合拿第一批熟人样本，不适合替代全部渠道判断。
