# 任务包模板

最后更新：2026-03-29
状态：现行标准件
适用阶段：`V4-Trial`
说明：本模板以当前仓为唯一现行落点；用于新任务的最小起包，不依赖旧仓路径

## 一句话目标

把新任务从“口头描述”变成“可继续、可恢复、可验收”的标准任务包。

## 使用步骤

1. 先确定任务 ID，格式必须为 `v4-trial-<三位序号>-<语义名>`。
2. 在 `.codex/chancellor/tasks/<task-id>/` 下创建 5 个标准文件。
3. 在 `active-task.txt` 中写入当前激活任务 ID。
4. 按推进过程持续更新 `state.yaml`、`decision-log.md`、`gates.yaml`、`result.md`。

## 目录模板

```text
.codex/chancellor/
  active-task.txt
  tasks/
    <task-id>/
      contract.yaml
      state.yaml
      decision-log.md
      gates.yaml
      result.md
```

## `active-task.txt` 模板

```text
v4-trial-002-task-package-template
```

## `contract.yaml` 模板

```yaml
task_id: v4-trial-xxx-your-task
title: 任务标题
goal: >-
  用一句完整的话写清任务要达成什么结果。
constraints:
  - 不得突破的边界 1
  - 不得突破的边界 2
acceptance:
  - 可验收标准 1
  - 可验收标准 2
must_gate:
  - 需要主公拍板的事项；若无则写 []
default_auto:
  - 可默认自动推进的低风险动作
source_refs:
  - 优先写当前目录内文件路径
```

## `state.yaml` 模板

```yaml
task_id: v4-trial-xxx-your-task
status: drafting
risk_level: low
next_action: 下一步最小动作
blocked_by: []
updated_at: '2026-03-29 00:07:51'
phase_hint: optional_phase
```

## `decision-log.md` 模板

```md
# 决策记录

## 2026-03-29 00:07:51

- 决策：本次关键决定
- 原因：为什么这样做
- 证据：依据了哪些当前目录内文档或结果
- 影响：后续会影响什么
```

## `gates.yaml` 模板

```yaml
task_id: v4-trial-xxx-your-task
items: []
```

## `result.md` 模板

```md
# 结果摘要

## 已完成

- 已完成事项 1
- 已完成事项 2

## 验证证据

- 文档或文件路径 1
- 文档或文件路径 2

## 遗留事项

- 尚未完成事项 1
- 尚未完成事项 2
```

## 填写原则

- 目标、边界、验收必须写具体，不写空话。
- `source_refs` 优先引用当前目录内文件，不直接依赖旧仓路径存活。
- `gates.yaml` 无待拍板时也必须显式写 `items: []`。
- `result.md` 不能只写结论，必须写验证证据与遗留事项。

## 与公开仓的关系

- 本文档可进入公开仓，作为固定模板说明。
- 真实运行态任务包继续只留本地，不进入公开仓。
- 公开仓中的 `.codex/chancellor/tasks/README.md` 只保留结构说明，不放真实任务内容。
