# V4-Trial 缺陷清单

最后更新：2026-03-29
状态：现行验收材料
适用阶段：`V4-Trial`

## 一句话结论

当前 `V4-Trial` 已能试跑和恢复，但仍存在若干工程化缺口；这些缺口不会否定 Trial 成立，但会影响升格后的稳定性。

## P1 缺陷

### 1. `git index.lock` 偶发残留

- 现象：多次提交时出现 `.git/index.lock` 冲突，需要串行清锁再提交。
- 影响：降低连续自动推进的稳定性。
- 当前状态：已形成现行处置方案，见 `docs/40-执行/10-本地安全提交流程.md`；但尚未工程化根治。

## P2 缺陷

### 3. 任务包模板仍偏文档化，未形成半自动脚手架

- 现象：新任务包仍依赖人工创建 5 件套文件。
- 影响：规模扩大后，重复劳动会增加。
- 当前状态：已有模板，但未自动化。

### 4. 执行区证据稿仍保留在现行目录内

- 现象：时间戳过程稿目前仍在 `docs/40-执行/`。
- 影响：随着文件增多，仍可能干扰阅读。
- 当前状态：已有角色说明，但尚未进入归档清理阶段。

## 已在本轮补齐的问题

### 4. Trial 汇总入口同步

- 说明：07/08/09 三份材料已纳入总览、首页、docs 入口与执行区入口。
- 结论：该问题已在本轮解决，不再作为待修缺陷。
