# 任务包规范

最后更新：2026-03-29
状态：现行标准件
适用阶段：`V4-Trial`
说明：本规范以当前仓为唯一现行落点；外部材料仅作历史参考，不构成运行依赖

## 一句话目标

任务包用于承载单条真实任务的合同、状态、决策、拍板与结果，运行态不再写进 `docs/`，并且整套任务包规则必须保持当前目录内自含、可迁移。

## 根目录

```text
.codex/chancellor/
  active-task.txt
  tasks/
    <task-id>/
```

## 任务 ID 规则

- 统一格式：`v4-<trial/target>-<三位序号>-<语义名>`
- 示例：`v4-trial-001-mvp-boundary-freeze`, `v4-target-001-baseline-freeze`
- 一个任务 ID 终生不复用，只归档、不覆盖

## 独立可迁移原则

- 现行任务包规范、模板、样例与运行态都以当前目录为落点。
- 任务继续推进时，不应依赖旧仓中的实时文件路径或外部目录结构。
- 若需要吸收旧仓知识，应先提炼并落到当前目录，再继续引用当前目录文件。
- 公开仓只放公开安全文档；真实运行态继续只留本地，但结构定义必须在当前目录内完整可见。
- 使用脚手架起包后，默认应回到 `docs/40-执行/14-维护层动作矩阵与收口检查表.md` 完成收口。

## 标准文件组

每个任务包固定包含以下 5 个文件：

- `contract.yaml`：任务目标、边界、验收、来源
- `state.yaml`：当前状态、风险、下一步、阻塞、更新时间
- `decision-log.md`：关键决策、原因、证据、影响
- `gates.yaml`：待拍板事项；若无则写 `items: []`；如存在待拍板项，字段结构以 `docs/40-执行/15-拍板包准备与收口规范.md` 为准
- `result.md`：当前结果、验证证据、遗留事项

## `active-task.txt`

- 只放一行纯文本 `task-id`
- 若为空，表示当前无默认激活任务
- 面板命令在未显式指定任务时，优先读取这里

## `contract.yaml` 最小字段

- `task_id`
- `title`
- `goal`
- `constraints`
- `acceptance`
- `must_gate`
- `default_auto`
- `source_refs`（优先引用当前目录内文件；若来源于旧仓，必须先在当前目录形成提炼稿后再引用）
- `planning_hint`（可选；用于记录当前主假设与最小推进策略入口）

## `state.yaml` 最小字段

- `task_id`
- `status`：`drafting / ready / running / waiting_gate / waiting_assist / verifying / done / paused / ready_to_resume`
- `risk_level`：`low / medium / high / critical`
- `next_action`
- `blocked_by`
- `updated_at`
- `phase_hint`（可选）
- `plan_step`（可选；用于记录当前最小推进步）
- `verify_signal`（可选；用于记录当前步完成后要看什么信号）

补充规则：当多个 gate 或多个异常并存时，`state.yaml` 只允许保留一个主状态；裁决规则以 `docs/40-执行/19-多 gate 与多异常并存处理规则.md` 为准。

## 生命周期

1. 创建任务包
2. 写合同
3. 若任务不确定性较高，先写主假设与最小推进步
4. 按收口检查表确认起包已闭环
5. 推进状态
6. 记录决策
7. 按 `docs/40-执行/15-拍板包准备与收口规范.md` 形成拍板包
8. 写结果与验证
9. 任务完成后保留目录作复盘证据

## 规划策略补充规则

- 若任务不确定性较高，可按 `docs/30-方案/07-V4-规划策略候选规范.md` 在现有 5 件套内记录主假设、最小推进步与验证信号。
- 主假设优先落在 `contract.yaml` 的 `planning_hint`。
- 当前最小推进步与验证信号优先落在 `state.yaml` 的 `plan_step` 与 `verify_signal`。
- 路线修正、失败原因与改路依据继续落在 `decision-log.md`。
- 本轮验证结论与下一步建议继续落在 `result.md`，不新增第 6 个文件。

## 当前特别约束

- 当前不承诺 `archive/` 子目录。
- 运行态只留本地，不推公开仓。
- 公开仓只提交公开安全文档与说明文件。
- 旧 `docs/.../v4-state/` 不再作为现行运行态。
- 当前目录应能独立承载现行规则与成品，不把关键定义散落到旧仓或外部目录。
