# V4 下一阶段优化清单

最后更新：2026-04-05
状态：现行执行件
适用范围：`丞相模式` 本体仓、`codex-home-export/`、维护层治理与后续反哺式优化
说明：本清单只写当前阶段值得继续做的少量优化，不把丞相模式重新拉回“大平台开发”路线。

## 一句话结论

当前阶段不再继续“大开发”，只做 3 类优化：处理真实稳定性问题、降低状态漂移、从真实任务里反向提炼少量复用件。

## 使用原则

1. 先看 `docs/40-执行/36-V4-丞相模式封顶边界清单.md`，再按本清单决定是否开下一刀。
2. 一次只开 1 个优化项，不并行铺摊子。
3. 每个优化项都必须写清：改哪些文件、怎么验收、做完算什么状态。
4. 若某项优化不能直接提高业务项目开发效率，就先不做。

## 推荐执行顺序

1. 先做 `git index.lock` 偶发残留治理。
2. 再做 `当前真实状态总表` 半自动追平。
3. 最后只在真实业务任务里反复出现同一痛点时，再补 `1-2` 个高频复用件。
4. 其余事项全部延后，不提前开坑。

## 每项开工前必须回答的 3 个问题

1. 这件事是不是已经在真实任务里重复出现，而不是一时想法。
2. 这件事做完后，能不能直接提高业务项目开发效率。
3. 这件事会不会打乱 `传令：XXXX` 单入口或增加明显维护负担。

如果有任意一个问题答不上来，就先不开工。

## 阶段完成线

当下面 4 条同时成立时，可视为“下一阶段优化已收口”：

1. `git index.lock` 不再是高频中断点，或已有稳定工程化兜底。
2. `当前真实状态总表` 在重装、重验后不再靠人工手补关键字段。
3. 高频复用件没有失控扩张，仍保持少量、可验证、可维护。
4. 没有重新滑回多 Agent、大平台、独立面板、海量 Skills 路线。
