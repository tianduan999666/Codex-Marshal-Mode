最后更新：2026-04-02

适用范围：第一轮公开试用的 V2EX 首发
说明：目标不是继续改文案，而是给主公一份今天就能直接发的定稿与跟进卡片

## 一句话结论

如果第一轮先发 `V2EX`，就不要再临场改。直接用这一版标题和正文发出去；发完后只按固定动作记反馈、接回复、做 24 小时判断。

## 先记住这 4 条

1. 第一轮只测一个主渠道，先不分心。
2. 首帖先讲结果，不先讲内部体系。
3. 主入口始终写成官方 Codex 面板，不写回 Claude。
4. 帖子发出后，先收事实，不先急着改文案。

## 回复分流卡

### 对方说“我想试”

直接回：

```text
好呀，我这套现在先支持 Windows 开发者试用。
你如果方便，我先把仓库和最短上手路径发你：
1. 先跑初始化
2. 必要时验真一下
3. 再建一个任务，然后回官方 Codex 面板直接继续

你试的时候如果卡住，直接把报错或卡点发我就行，我会按第一轮试用帮你一起走通。
```

### 对方说“看起来不错，但我现在没空”

直接回：

```text
没关系，这一轮我主要是想看第一上手路径哪里最重。
你如果现在没空，后面想试时直接回我一句也行；
我这边已经把初始化、建任务、回 Codex 面板继续这条线压到最短了。
```

### 对方明显不适配

直接回：

```text
感谢关注。
我这一轮先只收 Windows 开发者的真实试用反馈，
这样我能先把第一条上手路径压稳。
后面如果扩到别的平台，我再继续放开。
```
