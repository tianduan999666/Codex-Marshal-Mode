# V4 高频复用件总表

最后更新：2026-04-05
状态：现行执行件
适用范围：官方 Codex 面板里的日常开发、收口、切聊天与维护动作
说明：本表只保留当前真实高频、已经落到 AGENTS 或脚本链路里的复用件，不做“大而全技能库”。

## 一句话结论

当前只冻结 6 个高频复用件：最小闭环推进、推进汇报、缺陷修复、收口落盘、交班接班、维护验真。它们都直接服务“更稳、更好推进、更便于查看沟通、更有进度感”这条主线。

## 总表

| 复用件 | 什么时候触发 | 固定动作 | 当前落地 |
| --- | --- | --- | --- |
| 最小闭环推进件 | 用户直接下自然语言开发任务时 | 先判断瓶颈，再给最小推进步、验证信号，然后实现 | 已写入 `AGENTS.md` / `codex-home-export/AGENTS.md` |
| 推进汇报件 | 用户说“继续”“继续当前任务”，或同一主线连续多轮推进时 | 先用 4-6 行小表格说清 `已完成 / 当前瓶颈 / 当前最小推进点 / 验证状态 / 是否需要拍板`，再继续执行 | 已写入 `AGENTS.md` / `codex-home-export/AGENTS.md` |
| 缺陷修复件 | 用户说“修一下登录页”“有报错”“页面不对”这类修复任务时 | 优先按“定位最小复现 → 锁边界 → 最小修补 → 跑相关测试 → 收口留痕”推进 | 已写入 `AGENTS.md` / `codex-home-export/AGENTS.md` |
| 收口落盘件 | 用户说“收口”“交卷”“这一刀已完成”时 | 固定按 `已完成 / 结果 / 下一步` 汇报，并同步任务包与必要验证 | 已写入 `AGENTS.md` / `codex-home-export/AGENTS.md` |
| 交班接班件 | 用户准备切聊天框，或新聊天想无缝续上时 | `传令：交班` 生成 `progress-snapshot.md / handoff.md`；`传令：接班` 优先读取并续上 | 已落地 `invoke-panel-command.ps1`、`sync-task-context.ps1`、相关现行文档 |
| 维护验真件 | 任务涉及安装、升级、自检、受管文件改动、版本漂移时 | 固定执行“bump 版本 → 重装/重同步 → 验真 → 冒烟”闭环 | 已写入 `AGENTS.md` / `codex-home-export/AGENTS.md` 与维护链文档 |

## 使用原则

1. 复用件只能来自真实高频任务，不能为了“看起来像平台”先造一堆抽象壳。
2. 复用件必须能直接减少沟通成本、减少重复动作或提高连续推进能力。
3. 复用件一旦落地，优先写进 `AGENTS.md` 或脚本链，不只停留在方案文档。
4. 若一个复用件连续多轮都没实际用上，就不应继续扩写成子系统。

## 当前不做

- 不做独立 Skills 市场。
- 不做几十上百个“理论上也许会用到”的模板件。
- 不做脱离真实任务样本的“先抽象后落地”平台化工程。
