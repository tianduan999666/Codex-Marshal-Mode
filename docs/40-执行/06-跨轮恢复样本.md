# 跨轮恢复样本

最后更新：2026-03-29
状态：现行样本说明
适用阶段：`V4-Trial`
说明：本样本用于演示如何从现有任务包中恢复一条任务，不依赖聊天上下文

## 样本任务

- `v4-trial-004-current-standard-entry-unification`

## 为什么选它

- 已完成闭环，证据完整。
- 涉及多文件入口统一，能验证恢复后的理解是否完整。
- 风险低，适合作为第一份恢复样本。

## 恢复演示顺序

1. 读取 `.codex/chancellor/tasks/v4-trial-004-current-standard-entry-unification/state.yaml`
2. 确认状态为 `done`，下一步为规划第二条真实试跑任务。
3. 读取 `contract.yaml`，确认该任务目标是统一现行标准件入口。
4. 读取 `decision-log.md`，确认选择该任务的原因是“短期稍费力、长期收益更高”。
5. 读取 `result.md`，确认已完成事项、验证证据与遗留事项。

## 恢复后的结论示例

- 当前任务已完成，不需要继续执行本任务本身。
- 后续应转到下一条真实试跑任务，而不是重新讨论入口统一是否要做。
- 若需要复盘，只需回看该任务包，不必回翻整段聊天历史。
