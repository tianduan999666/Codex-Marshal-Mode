# 面板人工验板打勾单

适用场景：`verify-cutover.ps1` 已通过，准备进入官方 `Codex` 面板做最终人工验板时使用。

## 开始前

- [ ] 已先执行 `codex-home-export/verify-cutover.ps1`
- [ ] 已关闭旧 `Codex` 会话
- [ ] 已打开官方 `Codex` 面板
- [ ] 已新开一个全新会话
- [ ] 已额外验证自然语言入口：`传令：测试入口是否稳态`

## 第一关：`传令 版本`

- [ ] 命令能正常响应
- [ ] 能说清当前版本号
- [ ] 能说清版本来源
- [ ] 能说清真源路径是 `codex-home-export/VERSION.json`

## 第二关：`传令 检查`

- [ ] 命令能正常响应
- [ ] 能说明检查范围
- [ ] 能用人话给出检查结论
- [ ] 能给出下一步建议或引导到修复 / 验板

## 第三关：`传令 状态`（需要时再验）

- [ ] 命令能正常响应
- [ ] 能说清当前模式
- [ ] 能说清现在是否稳态
- [ ] 能给出最小下一步建议

## 最终判断

- [ ] 整个过程没有明显崩溃、跑题、失焦
- [ ] 整个过程不需要手改本地文件
- [ ] 我判断：这次人工验板通过

## 如果不通过

- [ ] 先重跑 `codex-home-export/verify-cutover.ps1`
- [ ] 若仍异常，再执行 `codex-home-export/rollback-from-backup.ps1`
- [ ] 回退后重新打开官方面板，再次验板
- [ ] 记录本轮失败点，不带着问题进入真实任务试跑

## 最短顺序

1. `传令：测试入口是否稳态`
2. `传令 版本`
3. `传令 检查`
4. `传令 状态`（必要时）
