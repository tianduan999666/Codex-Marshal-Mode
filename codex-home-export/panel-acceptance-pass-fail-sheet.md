# 面板人工验板打勾单

适用场景：`verify-cutover.ps1` 已通过，准备进入官方 `Codex` 面板做最终人工验板时使用。

## 开始前

- [ ] 已先执行 `codex-home-export/verify-cutover.ps1`
- [ ] 已关闭旧 `Codex` 会话
- [ ] 已打开官方 `Codex` 面板
- [ ] 已新开一个全新会话
- [ ] 已看到示例：`例如：传令：计算1+1=?`
- [ ] 已额外验证自然语言入口：`传令：测试入口是否稳态`

## 第一关：开场白与边界

- [ ] 命令能正常响应
- [ ] 开场白固定为 `🪶 军令入帐。亮，即刻接管全局。`
- [ ] 能明确说明只检查丞相自己，不检查你的项目
- [ ] 没有回退为旧命令体系

## 第二关：`传令：版本`

- [ ] 命令能正常响应
- [ ] 能说清当前版本号
- [ ] 能说清版本来源
- [ ] 能说清真源路径是 `codex-home-export/VERSION.json`

## 第三关：`传令：状态`

- [ ] 命令能正常响应
- [ ] 能说清 `版本`
- [ ] 能说清 `上次检查`
- [ ] 能说清 `自动修复`
- [ ] 能说清 `关键文件一致性`
- [ ] 能说清 `当前模式`
- [ ] 能说清 `当前任务`

## 第四关：`传令：升级`（如有执行）

- [ ] 命令能正常响应
- [ ] 能明确说明升级要由用户主动提出
- [ ] 能明确说明不会擅自升级用户项目

## 最终判断

- [ ] 整个过程没有明显崩溃、跑题、失焦
- [ ] 整个过程不需要手改本地文件
- [ ] 我判断：这次人工验板通过

## 如果不通过

- [ ] 先重跑 `codex-home-export/verify-cutover.ps1`
- [ ] 若仍异常，再执行 `codex-home-export/rollback-from-backup.ps1`
- [ ] 回退后重新打开官方面板，再次验板
- [ ] 记录本轮失败点，不带着问题进入真实任务试跑

## 最短顺序

1. `传令：测试入口是否稳态`
2. `传令：版本`
3. `传令：状态`
4. `传令：升级`（必要时）
