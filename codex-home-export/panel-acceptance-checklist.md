# 面板人工验板清单

适用场景：完成 `install-to-home.ps1` 与 `verify-cutover.ps1` 后，人工确认本机生产切换是否丝滑。
当前验板命令口径以 `codex-home-export/VERSION.json` 的 `panel_commands` 为准。

## 帮助命令公开用法

- `丞相帮助`：显示当前用法、命令与注意事项。

## 帮助命令固定结构

- `当前用法`：先说明当前默认入口与适用场景。
- `面板命令`：再列出当前面板丞相命令及其含义。
- `注意事项`：最后提示维护层动作、安全边界与人工验板提醒。

## 当前用法固定子项

- `默认入口`：当前默认用户入口是官方 Codex 面板。
- `适用场景`：普通用户日常使用优先在面板内完成，终端仅保留给维护、安装、迁移、排障。
- `维护层说明`：如必须落到脚本层，要明确说明“这是维护层动作”。

## 面板命令固定子项

- `命令顺序`：按丞相帮助、丞相状态、丞相检查、丞相修复、丞相验板、丞相版本的顺序展示。
- `命令含义`：每条命令都要带一句人话含义，不得只列命令名。
- `公开边界`：普通用户只暴露面板命令，不主动推荐终端丞相别名。

## 注意事项固定子项

- `安全边界`：普通用户入口优先留在官方面板，维护层动作不向普通用户外露复杂终端流程。
- `维护层动作`：如必须落到脚本层，要明确说明“这是维护层动作”。
- `新开会话验板提醒`：入口相关改动后，建议新开官方面板会话做人眼验板。

## 相关维护命令边界

- `丞相修复`：在安全边界内尝试自动修复常见问题。
- `丞相验板`：给出进入官方面板人工验收的固定步骤。

## 丞相修复固定槽位

- `修复范围`：只在安全边界内处理当前常见问题。
- `处理方式`：优先尝试自动修复当前已知常见问题。
- `升级条件`：超出安全边界或无法自动修复时停止扩展并提示人工处理。

## 丞相验板固定槽位

- `触发场景`：入口相关改动后或切换后进入官方面板做人眼验板。
- `验板动作`：给出进入官方面板人工验收的固定步骤。
- `验板目标`：确认版本、模式与入口表现是否稳态。

## 验板步骤

1. 关闭当前 `Codex` 会话。
2. 重新打开官方 `Codex` 面板，新开一个全新会话。
3. 首句输入：`丞相版本`
4. 继续输入：`丞相检查`
5. 如需再验一层，继续输入：`丞相状态`

## 验板步骤固定子项

- `关闭旧会话`：关闭当前 `Codex` 会话。
- `新开官方面板`：重新打开官方 `Codex` 面板，新开一个全新会话。
- `版本验证`：首句输入：`丞相版本`
- `检查验证`：继续输入：`丞相检查`
- `状态验证`：如需再验一层，继续输入：`丞相状态`

## 丞相版本固定槽位

- `版本号`：返回当前丞相模式版本号。
- `版本来源`：明确当前版本来自哪个版本来源。
- `真源路径`：优先说明当前版本真源路径为 `codex-home-export/VERSION.json`。

## 丞相检查固定槽位

- `检查范围`：只做最小必要检查，不扩展成无关大扫除。
- `检查结论`：用人话汇报当前是否存在明显异常或风险。
- `建议动作`：如发现问题，给出下一步建议或引导到修复 / 验板。

## 丞相状态固定槽位

- `当前模式`：明确当前处于什么模式或入口状态。
- `稳态判断`：明确当前是否稳态。
- `下一步`：给出最小下一步建议。

## 通过标准

- `丞相版本` 能返回当前丞相模式版本与版本来源。
- `丞相版本` 的输出能覆盖“版本号 → 版本来源 → 真源路径”。
- `丞相检查` 能做最小必要检查并返回人话结论。
- `丞相检查` 的输出能覆盖“检查范围 → 检查结论 → 建议动作”。
- `丞相状态` 能汇报当前模式、是否稳态、下一步。
- `丞相状态` 的输出能覆盖“当前模式 → 稳态判断 → 下一步”。
- `丞相修复` 的边界说明能覆盖“修复范围 → 处理方式 → 升级条件”。
- `丞相验板` 的边界说明能覆盖“触发场景 → 验板动作 → 验板目标”。
- 人工验板步骤能覆盖“关闭旧会话 → 新开官方面板 → 版本验证 → 检查验证 → 状态验证”。
- 整个过程不出现明显崩溃、失焦或命令失效。
- 整个过程无需再手改本地文件。

## 通过标准固定子项

- `命令有效`：版本、检查、状态命令可用且口径完整。
- `边界稳定`：修复与验板边界说明完整。
- `验板闭环`：人工验板步骤模板完整。
- `过程稳定`：整个过程不出现明显崩溃、失焦或命令失效。
- `无需手改`：整个过程无需再手改本地文件。

## 若不通过

1. 先执行：`codex-home-export/verify-cutover.ps1`
2. 若仍异常，再执行：`codex-home-export/rollback-from-backup.ps1`
3. 回退后重新打开面板，再次验板。

## 若不通过固定子项

- `自动复核`：先执行：`codex-home-export/verify-cutover.ps1`
- `自动回退`：若仍异常，再执行：`codex-home-export/rollback-from-backup.ps1`
- `重新验板`：回退后重新打开面板，再次验板。
