## 现在默认怎么开工

- 第一次准备环境，仍用维护层脚本。
- 日常开工，优先回官方 Codex 面板直接说：`传令：我要做 XX`。
- 面板内会先走 `codex-home-export/start-panel-task.ps1`。
- 系统对外固定流程是：`先确认丞相能正常接到传令 → 再确认丞相自身状态良好 → 接着把丞相调整到最佳工作状态 → 丞相记录这次要做的任务 → 丞相开始执行任务`。
- 系统内部仍会按最小必要原则做轻量检查、必要时完整验真与自动修复；通过后写入本机验真状态。
- 后续同版本任务默认跳过重复验真，直接建任务。
- 用户可主动查询：`传令：状态 / 传令：版本 / 传令：升级`。
- `传令：状态` 固定优先展示六行：`版本 / 上次检查 / 自动修复 / 关键文件一致性 / 当前模式 / 当前任务`。
- `传令：升级` 只在用户主动提出时处理，系统默认不自动升级。
- 固定边界提示是：`提示：丞相在检查阶段只检查自己，不会查看你的项目；执行阶段只按你的传令办事，不会擅自审查项目。`
- 成功后留在当前会话继续，不再先把你赶去 PowerShell。
